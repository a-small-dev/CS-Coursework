# Software Development Lifecycle (SDLC) Coursework

This repository contains coursework and assignments completed for my Software Development Lifecycle class as part of my SNHU Computer Science studies.

## Overview

Throughout this course, I explored the principles and practices of modern software development processes, with a strong focus on **Agile methodologies**, particularly **Scrum**. The course emphasized how teams plan, build, and deliver software in iterative cycles while adapting to change and improving continuously.

## What I Learned

- Fundamentals of the Software Development Lifecycle (SDLC)
- Differences between traditional (Waterfall) and Agile approaches
- Core Scrum concepts, including:
  - Sprints and sprint planning
  - Product backlogs and user stories
  - Daily stand-ups
  - Sprint reviews and retrospectives
- Team roles in Scrum:
  - Product Owner
  - Scrum Master
  - Development Team
- Importance of collaboration, communication, and iterative development
- How Agile improves flexibility and delivery in software projects

## Key Takeaways

This course helped me understand how real-world software teams operate beyond just writing code. I gained knowledge in structured teamwork, iterative planning, and adapting to changing requirements. The skills that are essential in modern software development environments.

## Purpose of This Repository

This repository serves as a record of my learning and practical exercises related to SDLC concepts and Agile/Scrum methodology.
